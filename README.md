# Bus Tracking System Database

## Overview
This project implements a relational database for a simple **Bus Tracking System** using PostgreSQL. The objective of the project is to design a structured database that manages users, roles, and system data while maintaining proper relationships, data integrity, and access control.

The project demonstrates core **DBMS concepts** such as relational schema design, normalization, and role-based permissions.

---

## Features
- Relational database schema for a bus tracking system
- Tables with **Primary Keys** and **Foreign Key relationships**
- **Role-based access control** using PostgreSQL roles and privileges
- SQL queries using **JOINs, filtering, and aggregation**
- Normalized database structure to ensure data consistency

---

## Technologies Used
- **PostgreSQL**
- **SQL**

---

## Key Concepts Practiced
- Database schema design
- Table relationships (**Primary Key / Foreign Key**)
- SQL joins (**INNER JOIN**)
- Data filtering and grouping (**WHERE, GROUP BY**)
- User roles and privileges in PostgreSQL
- Basic database normalization

---

## Purpose
This project was developed as part of a **DBMS coursework project** to understand how relational databases are designed and managed in real-world systems such as transportation and tracking platforms.
